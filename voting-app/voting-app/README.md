# แอปโหวต (Voting App)

แอปโหวตแบบใช้จอเดียวร่วมกัน: ผู้ดูแลระบบสร้างรายการโหวตพร้อมตัวเลือกที่กำหนดเอง (ชื่อ, อิโมจิ, รูปภาพ, คำอธิบาย, สี) ผู้ใช้แตะเพื่อโหวต และผู้ดูแลระบบดูผลโหวตแบบเรียลไทม์ได้

## หน้าเพจ

- **แดชบอร์ด** (`/`) — รายการโหวตทั้งหมด กดเพื่อโหวต หรือใช้ไอคอน ✏️ แก้ไข / 👁️ ดูผลโหวต / 🗑️ ลบ (ทุกอย่างยกเว้นโหวต ต้องใช้รหัสผ่านผู้ดูแลระบบ)
- **แก้ไข** (`/edit/new` หรือ `/edit/:id`) — ตั้งชื่อรายการและสร้างตัวเลือก
- **โหวต** (`/vote/:id`) — แตะตัวเลือกเพื่อโหวต ไม่แสดงคะแนนในหน้านี้ ใช้ซ้ำได้หลายครั้งจากเครื่องเดียวกัน มีเอฟเฟกต์ประกายดาว ✨ ยืนยันเมื่อโหวตสำเร็จ
- **ผลโหวต** (`/score/:id`) — ต้องใส่รหัสผ่านผู้ดูแลระบบ (ตั้งค่าเป็น `admin`) แสดงผลแบบกราฟแท่ง มีปุ่มรีเฟรชและรีเซ็ตคะแนน

## รหัสผ่านผู้ดูแลระบบ

รหัสผ่านเดียว (`admin`) ใช้ป้องกันทุกอย่าง: แก้ไข, ลบ, และดูผลโหวต ตั้งค่าอยู่ที่ไฟล์เดียว:

```
src/constants.js
```

```js
export const ADMIN_PASSWORD = 'admin'
```

## 1. ตั้งค่า Firebase

1. ไปที่ [Firebase console](https://console.firebase.google.com/) แล้วสร้างโปรเจกต์ใหม่ (หรือใช้โปรเจกต์เดิม)
2. เพิ่ม **Web app** ในโปรเจกต์ (ไอคอน `</>`) แล้วคัดลอกค่า config ที่ได้
3. คัดลอกไฟล์ `.env.example` เป็น `.env` แล้ววางค่าที่ได้ลงไปในนั้น (ไฟล์ `.env` ถูกใส่ไว้ใน
   `.gitignore` แล้ว จึงไม่ถูกอัปโหลดขึ้น GitHub):

   ```bash
   cp .env.example .env
   ```

   ```
   VITE_FIREBASE_API_KEY=xxxxxxxx
   VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
   VITE_FIREBASE_PROJECT_ID=your-project-id
   VITE_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
   VITE_FIREBASE_MESSAGING_SENDER_ID=xxxxxxxx
   VITE_FIREBASE_APP_ID=xxxxxxxx
   ```

   ⚠️ **เรื่อง API key กับ GitHub**: ไฟล์ `.env` จะไม่ถูกอัปโหลดขึ้น repo (ถูก ignore ไว้แล้ว)
   แต่ที่จริง Firebase web API key ไม่ใช่ความลับแบบ API key ฝั่งเซิร์ฟเวอร์ทั่วไป — มันแค่บอกว่า
   request นี้เป็นของโปรเจกต์ไหน ตัวที่ป้องกันข้อมูลจริงๆ คือ **Firestore security rules** (ดูด้านล่าง)
   ถ้า rules เปิดโล่ง (`allow read, write: if true`) ใครก็ตามที่รู้ค่า config จะอ่าน/เขียนฐานข้อมูลได้หมด
   ไม่ว่าคีย์จะถูกซ่อนไว้หรือไม่ก็ตาม ดังนั้นสิ่งสำคัญกว่าการซ่อนคีย์คือการตั้ง rules ให้รัดกุมถ้าจะเปิดใช้งานจริงในระยะยาว
4. เปิดใช้งาน **Firestore Database** (Build → Firestore Database → Create database) เริ่มด้วย test mode หรือใช้ rules ด้านล่าง
5. เปิดใช้งาน **Storage** (Build → Storage → Get started)

   ⚠️ **สำคัญ**: ตั้งแต่ปลายปี 2024 เป็นต้นมา Firebase กำหนดให้ต้องอัปเกรดโปรเจกต์เป็นแผน **Blaze (pay-as-you-go)**
   ก่อนถึงจะเปิดใช้งาน Cloud Storage ได้ (ยังมี free tier ให้ใช้ฟรีตามโควตาอยู่ ไม่ได้แปลว่าต้องเสียเงินทันที)
   ถ้ายังเป็นแผน Spark (ฟรี) อยู่ การอัปโหลดรูปจะล้มเหลวเงียบๆ — ปัญหานี้เป็นสาเหตุที่พบบ่อยที่สุดที่ทำให้ตัวเลือกที่มีรูปภาพบันทึกไม่สำเร็จ

### Firestore rules (สำหรับทดสอบ/ใช้งานทั่วไป — แอปนี้ไม่มีระบบล็อกอิน)

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /programmes/{programmeId} {
      allow read, write: if true;
      match /choices/{choiceId} {
        allow read, write: if true;
      }
    }
  }
}
```

### Storage rules (สำหรับทดสอบ/ใช้งานทั่วไป)

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /voting-app/{allPaths=**} {
      allow read, write: if true;
    }
  }
}
```

⚠️ กฎเหล่านี้เปิดให้ใครก็ตามที่มีค่า config ของโปรเจกต์อ่าน/เขียนข้อมูลได้ เหมาะสำหรับแอปใช้งานกันเองแบบสบายๆ
แต่ถ้าจะเปิดให้คนทั่วไปใช้งานจริงในระยะยาว ควรเพิ่มระบบยืนยันตัวตนและจำกัดสิทธิ์ให้รัดกุมกว่านี้

## 2. ติดตั้งและรัน

```bash
npm install
npm run dev
```

แล้วเปิดลิงก์ที่ขึ้นมาในเทอร์มินัล

## แก้ปัญหา: เพิ่มตัวเลือกแล้วไม่เข้าฐานข้อมูล

ระบบตอนนี้บันทึกแต่ละตัวเลือกแยกกัน ถ้าตัวเลือกหนึ่งมีปัญหา (เช่น อัปโหลดรูปไม่สำเร็จ) ตัวเลือกอื่นจะยังถูกบันทึกอยู่
และหน้าจอจะแจ้งเตือน (⚠️) ว่าตัวเลือกไหนมีปัญหาและเพราะอะไร ให้ตรวจสอบตาม 2 จุดนี้ก่อน:

1. **Storage เปิดใช้งานหรือยัง** และอัปเกรดเป็นแผน Blaze แล้วหรือยัง (ดูหัวข้อด้านบน)
2. **Storage rules** ตั้งค่าให้อ่าน/เขียนได้ตามตัวอย่างด้านบนหรือยัง
